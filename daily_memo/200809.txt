프론트엔드 개발 올인원 패키지 with React Online.를 결제 했다.

github 따라만들기 예제 실습 1~10

- meta, 반응형 사이트를 만들기 위해서는 viewport를 설정해줘야 한다.
    -> <meta name="viewport" content="width=device-width, initial-scale=1,                           user-scalable=no,                           maximum-scale=1, minimum-scale=1">
    ->                     너비는 기기의 너비를 그대로 사용, 최초의 스케일은 확대축소를 원하지 않으면 1,   사용자가 확대 축소하는 동작을 막을때는 no,     만약 확대, 축소를 허용하였다면 최대 확대 비율, 최소 축소비율을 지정할 수 있다.

- meta, <meta http-equiv="X-UA-Compatible" content="ie=edge">
    -> 인터넷 익스플로러의 랜더링을 최신버전으로 하게 설정하는 부분.

- meta, <meta property="og:type" content="website">
    -> og 는 open graph 의 약자이고 이 사이트의 대표적인 정보들을 정의해주는 태그입니다.
- meta, <meta property="twitter:card" content="summary">
    -> twitter card 도 og와 비슷한 기능을 하는 방식입니다. property 를 트위터 카드 형태로 정의 한다는 것.
==> og, twitter 의 정보는 공유할때 보여지는 부분들(카카오톡 공유시)

- meta, <link rel="apple-touch-icon" href="favicon.png">
    -> 스마트폰에서 사용하는 아이콘으로 이용할 수 있다. (안드로이드도 가능)
    -> 익스플로러 에서는 ico 파일을 사용하는데 루트에 favicon.ico 파일이 존재하면 굳이명시해 주지 않아도 적용이 됩니다.

- google font 사이트에서 가져온 폰트를 적용시켜 주었습니다. css 파일은 확장이 가능하게(어지럽지 않게) 폴더를 분류하여 정리해줍니다.
    ->    <link href="https://fonts.googleapis.com/css2?family=Roboto:wght@300;400;500&display=swap" rel="stylesheet">
          <link rel="stylesheet" href="css/main.css">

- reset.css 를 적용시켰다. 기본적으로 각각의 브라우저 등에서 사용되는 css 를 초기화 시켜주기 위함이다.
    -> reset.css 같이 외부에서 가져오는 라이브러리 같은 경우에는 내가 만들 코드보다 먼저 적용을 시켜주어야 한다.
    -> cdn 을 이용하면 매번 파일을 준비하지 않아도 됨

- <div class="body__container"> 콘테이너 클래스를 이렇게 선언 해주었는데 이는 BEM 작명 규칙을 사용.
    -> - 일반적인 작명
    -> __ ~의 일부분 ( 자식 )
    -> -- ~의 상태   ( btn--sucess, btn--danger ...)

- .input--text::-webkit-input-placeholder (webkit, ms, moz)등을 통해 공식적이지 않지만 실험적으로 사용 가능한 css 기능들을 사용해 볼수 있습니다.
